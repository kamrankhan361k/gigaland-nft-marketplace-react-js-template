How to Install
  1. Node.js and NPM : You can download Node.js from https://nodejs.org. please use version 18.14.2
  2. NPM : You must have to npm install packages. You can do this by running npm install from the root of your project to install all the necessary dependencies.

    Open a terminal(Windows Command Prompt or PowerShell).

  3. Run npm start for a dev server. Navigate to http://localhost:3000/ . The app will automatically reload if you change any of the source files.
  4. Run npm run build to build the project. The build artifacts will be stored in the build/ directory. Use the npm run build --prod flag for a production build.
If not work using NPM try using Yarn
  1. add SKIP_PREFLIGHT_CHECK=true in file .env
  2. Remove node module command rm -rf node_modules
  3. Delete package-lock.json
  4. Install Yarn ( https://yarnpkg.com/en/docs/install ).
  5. Run yarn install
  6. Run yarn run start from the folder
Install Gigaland-nft Minting with Metamask
  1. to start using this functionality you have to install and create an account for metamask, you will have to configure it to use the Goerli test network
  2. register an account on alchemy website and pinata first, you will have to use the API key and Secret to start using it - https://www.alchemy.com/ - https://app.pinata.cloud/
  3. in Alchemy, create new app with Goerli as the selected network
  4. click GET TEST ETH, then login alchemy in the website https://goerlifaucet.com/
  5. in goerlifaucet.com, paste Goerli test address from metamask to get some test ETH
  6. after installing the metamask follow this article to get your private key https://metamask.zendesk.com/hc/en-us/articles/360015289632-How-to-Export-an-Account-Private-Key
  7. after that, you can open the gigaland-nft folder, this folder is needed to create the smart contract that we will deploy to the ethereum network
  8. find the .env file, you will have to copy your alchemy API url and metamask private key
  9. install all the dependencies with npm install, then run npx hardhat compile
  10. run npx hardhat run scripts/deploy.js --network goerli
  11. you will see the contract address printed on the command line, keep this address so we can use it to mint our NFT
  12. open the gigaland folder, and find the .env file. we will copy the Pinata KEY and SECRET also the Alchemy API url into the file
  13. one last file need to be changed is the interact.js file inside src/core/nft/interact.js you will have to change this line `const contractAddress = "0x92d474d216ce429c1efc7f48d7ff66bae5d070cd";` with the address you keep before
  14. you can start developing and test the minting process by going to the minting page
  15. more references: - https://ethereum.org/en/developers/tutorials/how-to-mint-an-nft/
How to install strapi CMS in your local
  1. Node.js and NPM : You can download Node.js from https://nodejs.org. please use version 18.14.2.
  2. Enter to gigaland-strapi directory
  3. Run npm install
  4. After that, run npm run develop to deploy the cms project in your local directory
  5. Open http://localhost:1337/admin to enter the cms dashboard
  6. Change password strapi with new password in your profile setting

  7. Enter directory gigalandlightgrey / gigaland-retro / gigaland-blackyellow
  8. Run npm install
  9. After that, run npm start for work in local your directory at localhost:3000
  10. Run npm run build to build the project. The build artifacts will be stored in the build/ directory. Use build directory for upload in your server

Strapi CMS to DigitalOcean Droplet

Since Gigaland is using Node v18.14.2, DigitalOcean App deployment is not supported, instead deploying to Droplet

Create SSH key

  1. in local terminal/git bash, generate ssh key by typing ssh-keygen, set filename then give password
  2. copy the text of public key : cat id_rsa.pub
  3. in digitalocean, create ssh key in the setting, select secuirty tab, click add SSH Key, then paste public key text, give the name and click add SSH Key

Create and Access a Droplet in DigitalOcean

  1. create Droplet (shared VM) with strapi-capable spec (2GB, 1Core, 10GB)
  2. fill in all mandatory info, choose SSH key previously created
  3. after Droplet created, access the droplet with root : go to droplet detail, click Access, the Launch Droplet Console
  4. create non-root user, “This to set up a new system user account with reduced privileges, and configure this user to run administrative commands via "sudo"
  5. in the console type adduser <username>
  6. add the new user to the sudo group : usermod -aG sudo <username>
  7. connect the droplet, in terminal/git bash : ssh -i id_rsa <username>@<server ip address>

Setup Nginx as a Web Server

  1. in terminal/git bash type sudo apt update then sudo apt install nginx
  2. open firewall for HTTP : sudo ufw allow 'Nginx HTTP'
  3. check the firewall type sudo ufw status
  4. check the web server is running by typing systemctl status nginx

Installing Node with NVM

  1. setup curl if not existed : sudo apt install curl
  2. install node and type curl https://raw.githubusercontent.com/creationix/nvm/master/install.sh | bash
  3. make this accessible globaly : source ~/.profile
  4. setup Node Version Manager for easy version node changes : nvm install 18.14.2
  5. make sure the NVM using the version : nvm use v18.14.2
  6. and make it to default version if rebooted/restarted : nvm alias default v18.14.2

Transfer Project Files and Build

  1. Install Filezilla, create new connection, set the server IP, select logon type to Ask password, set username (non-root), and then connect and fill the password
  2. transfer the gigaland-strapi folder to the path /home/<username>/
  3. optionally you can transfer with other tools like WinSCP, Putty or cloning from GIT
  4. in terminal/git bash, connect to the droplet : ssh -i id_rsa <username>@<serverIpAddress>
  5. go the the project path, then npm install or yarn install (if yarn not existed : npm install yarn -g)
  6. build the project for production : NODE_ENV=production npm run build or NODE_ENV=production yarn build

Install and Configure PM2 (Process Manager) Runtime

“PM2 makes it possible to daemonize applications so that they will run in the background as a service.”

  1. edit this js file : nano ecosystem.config.js, then replace the text with following : 
      module.exports = {
    apps: [
      {
        name: 'strapi',
        cwd: '/home//gigaland-strapi',
        script: 'npm',
        args: 'start',
        env: {
          NODE_ENV: 'production',
          HOST:'127.0.0.1',
          PORT='1337',
        },
      },
    ],
  };
  2. enable firewall to the specific port since strapi listen port 1337 : sudo ufw 1337/tcp then sudo ufw enable
  3. install PM2 : sudo npm install pm2@latest -g
  4. check if the js file can run in background : pm2 start ecosystem.config.js
  5. make the js run in the startup : pm2 startup systemd
  6. run the command from the output : sudo env PATH=$PATH:/usr/bin /usr/lib/node_modules/pm2/bin/pm2 startup systemd -u username --hp /home/<username>
  7. save it : pm2 save
  8. finally reboot to check whether it’s starting and running : sudo reboot

Test and Create Admin Account

  1. in the browser, browse to <serverIpAddress>:1337/admin
  2. create admin account, set username and password
  3. use this url <serverIpAddress>:1337/admin in the api.js configuration file of Gigaland frontends
